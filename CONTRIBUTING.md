# Contributing to Agentic-AI Lab

Thank you for your interest in contributing to the Agentic-AI Lab project! We welcome contributions from the community.

## Ways to Contribute

### 1. Share Your Use Cases
Have you applied the Agentic-AI Framework to your domain? We'd love to hear about it!
- Open an issue describing your use case
- Share lessons learned and adaptations made
- Consider contributing a case study

### 2. Improve Documentation
- Fix typos or clarify explanations
- Add examples or tutorials
- Translate documentation to other languages
- Create diagrams or visualizations

### 3. Develop Tools and Code
- Implement framework components
- Create utilities for common tasks
- Develop integrations with existing tools
- Contribute benchmarks or evaluation scripts

### 4. Research Collaboration
- Validate the framework in new domains
- Propose extensions or improvements
- Conduct empirical studies
- Co-author papers

### 5. Community Building
- Answer questions in discussions
- Share the project with others
- Write blog posts or tutorials
- Present at meetups or conferences

## Getting Started

1. **Read the Documentation**
   - Start with [README.md](README.md)
   - Review [Agentic-AI Research Roadmap](docs/Agentic-AI-Research-Roadmap.md)
   - Check the [Timeline](docs/Research-Timeline-2025-2027.md)

2. **Check Existing Issues**
   - Look for issues tagged `good-first-issue` or `help-wanted`
   - Comment on issues you'd like to work on

3. **Join the Discussion**
   - Use GitHub Discussions for questions and ideas
   - Share your thoughts on open issues

## Contribution Guidelines

### For Documentation and Content

1. **Fork the repository**
2. **Create a branch** for your changes
   ```bash
   git checkout -b improve-documentation
   ```
3. **Make your changes**
   - Follow existing formatting and style
   - Use clear, concise language
   - Include examples where helpful
4. **Commit with descriptive messages**
   ```bash
   git commit -m "docs: clarify RAG pipeline section"
   ```
5. **Push and create a Pull Request**
   ```bash
   git push origin improve-documentation
   ```

### For Code Contributions

1. **Follow Python PEP 8** style guidelines (or relevant language conventions)
2. **Include docstrings** for functions and classes
3. **Add tests** for new functionality
4. **Update documentation** to reflect changes
5. **Keep commits atomic** - one logical change per commit
6. **Document decisions** - All significant PRs should include an update to `decision_log/` or `daily/` logs (following the Agentic Engineering ethos of persistent knowledge)

### Commit Message Convention

Use conventional commit format:
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `refactor:` - Code refactoring
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

Example: `feat: add vector database indexing utility`

## Code of Conduct

### Our Pledge

We are committed to providing a welcoming and inclusive environment for all contributors, regardless of:
- Experience level
- Background
- Identity
- Perspective

### Expected Behavior

- Be respectful and considerate
- Welcome diverse viewpoints
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior

- Harassment or discriminatory language
- Personal attacks or trolling
- Publishing others' private information
- Unethical or unprofessional conduct

### Enforcement

Instances of unacceptable behavior may be reported to bguan@fiu.edu. All complaints will be reviewed and investigated.
